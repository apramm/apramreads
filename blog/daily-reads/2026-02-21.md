- bipartite matching, matchings in bipartite graph can model situations in which **objects are being assigned to other objects**

- **determining largest matching in bipartite graph G**
    - oldest problem in combinatorial algos 
    - there is an algorithm which solves this problem in poly time but we need NETWORK FLOWS for that!

- Maximum-Flow problem
    - intially motivated by issue of traffic in network and diversified applications seen later

- **Flow Networks** this is a directed graph with the features : *capacities* on each edge (indicating how much they can carry i.e., is non-negative), *source nodes* in the graph which generate traffic; *sink* which "absorbs" traffic as it arrives, and finally traffic is transmitted across the edges between internal nodes
    - Flow is an abstract entity generated at source and transmitted across edges and absorbed by sink.
